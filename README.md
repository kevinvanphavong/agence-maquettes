# Maquettes de refonte — prospects

Démonstrations interactives de refonte de site web + back office, à présenter aux prospects.
En ligne : `https://kevinvanphavong.github.io/agence-maquettes/`

## Contenu du dossier

- `index.html` — page portfolio qui présente toutes les maquettes. **Protégée par un code d'accès** (garde-fou : réservée à l'administrateur).
- `kids-paradise-1994e1.html` — maquette Kid's Paradise
- `laser-game-8a5666.html` — maquette Laser Game Evolution Blois
- `fgc-b98b0a/` — maquette Family Games Center (site multi-pages, point d'entrée `fgc-b98b0a/index.html`)
- `inziair-d05f72.html` — maquette INZI'AIR
- `larecre-des-pirates-06ba89.html` — maquette La Récré des Pirates
- `jim-and-jump-a51a1f.html` — maquette Jim and Jump
- `play-n-jump-9d96bc.html` — maquette Play N'Jump
- `cap-karting-f8682f.html` — maquette Cap Karting
- `peka-bloc-d7473a.html` — maquette Peka Bloc

> **Pourquoi ces suffixes aléatoires ?** C'est volontaire et il ne faut PAS les « nettoyer ». Un prospect qui reçoit le lien de sa maquette ne doit pas pouvoir deviner l'URL de la maquette d'un autre prospect. Ne pas renommer ces fichiers avec des noms simples.

## Mettre à jour le site

Après toute modification, depuis le dossier :

```bash
cd ~/Desktop/agence-maquettes
git add -A
git commit -m "Mise à jour des maquettes"
git push
```

Le site en ligne se met à jour tout seul en moins d'une minute.
